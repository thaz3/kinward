# Screen 27 — Managed-minor profile

> **Draft high-fidelity design — not approved for implementation.**

**Viewport:** 390 × 844 mobile. **Source:** `06-minor-backup-and-continuity.md`, Screen 27. **Flow:** UF-16. **Tests:** AT-020, AT-043.

## Composition

Context: `Harbor Circle · Circle-wide`. Heading `Kai’s basic profile`; status badge with a person-and-lock icon plus `Managed profile`; supporting line `Managed by Morgan`. Fields: preferred name `Kai`, relationship `Family`, age band `Teen`, and a labeled visibility selection `Approved Circle members`. Information alert: `Age band changes display and age-appropriate experience only.` Primary `Save basic profile`; secondary `Cancel`.

Required visible boundary copy: `Kinward does not automatically convert a managed minor profile into an adult account. Minor-to-adult transition rules remain deferred.`

**State label:** Managed profile. **Primary action:** Save basic profile.

**Accessibility/privacy annotations:** Persistent labels and grouped visibility choices; no exact birth date; no sign-in, invitation, conversion, claim, ownership transfer, birthday invitation, adult role, permission, delegation, medical alert, or emergency alert affordance. Error summary retains authorized input.
